<h2 id="prompt-engineering-vs-context-engineering">Prompt Engineering vs. Context Engineering</h2><h3 id="prompt-engineering-%E7%9A%84%E4%BE%B7%E9%99%90">The Limits of Prompt Engineering</h3><p>When we first started communicating with LLMs, our primary focus was on <strong>Prompt Engineering</strong>.</p><p>The goal was simple: how do you write an effective prompt — especially a system prompt — so the LLM understands your instructions on the first try and produces the result you're after?</p><figure class="kg-card kg-bookmark-card"><a class="kg-bookmark-container" href="/prompt-engineering-four-essential-elements/"><div class="kg-bookmark-content"><div class="kg-bookmark-title">Top Teams' Prompt Engineering Secrets: Four Essential Elements to Master AI Prompting!</div><div class="kg-bookmark-description">This article synthesizes prompt engineering insights from GitHub, AWS, Anthropic, and Google. Through practical examples, it breaks down four core elements — task, context, persona, and format — to help you craft high-performance prompts that significantly improve AI comprehension, response accuracy, and output quality.</div><div class="kg-bookmark-metadata"><img class="kg-bookmark-icon" src="/images/ghost/icon/R0000314-6.webp" alt=""><span class="kg-bookmark-author">ReallyNiceDay</span><span class="kg-bookmark-publisher">Hao Tien Cheng</span></div></div><div class="kg-bookmark-thumbnail"><img src="/images/ghost/thumbnail/Untitled-2025-05-29-header.webp" alt="" onerror="this.style.display = 'none'"></div></a></figure><p>As LLMs have rapidly evolved, the tasks we delegate to AI have grown far more complex, requiring more background context and a wider set of tools along the way.</p><p>At this point, Prompt Engineering — which focuses on single, isolated interactions — gradually stops meeting our needs.</p><h3 id="context-engineering-%E7%9A%84%E5%B4%9B%E8%B5%B7">The Rise of Context Engineering</h3><figure class="kg-card kg-image-card"><img src="https://media.tenor.com/K-KRb8SDjhoAAAAC/solo-leveling.gif" class="kg-image" alt="" loading="lazy" width="498" height="282"></figure><p>To understand why, we first need to talk about AI Agents.</p><p>A simple definition of AI Agents:</p><blockquote>LLMs that actively reason, autonomously use tools (such as web search, file reading, and executing specific commands), and repeat this cycle until a goal is achieved — all without requiring step-by-step instruction for every action.</blockquote><p>Think of it like a human assistant. You don't need to spell out every detail upfront. The assistant uses their own judgment to look things up, use available tools, and find a way to accomplish the end goal you've given them.</p><p>As LLM capabilities have improved, Agents can now:</p><ul><li>Carry out more rounds of reasoning</li><li>Operate across longer time horizons</li><li>Execute more complex behaviors</li></ul><p>What we need is no longer just a technique for crafting a single instruction — we need a strategy for managing the entire interaction.</p><p><strong>Context Engineering</strong> emerged to fill that gap.</p><p>Whereas Prompt Engineering focuses on single, isolated tasks, Context Engineering operates iteratively across every exchange with the LLM.</p><h3 id="context-engineering-%E5%AF%A6%E9%9A%9B%E4%B8%8A%E5%A6%82%E4%BD%95%E9%81%8B%E4%BD%9C%EF%BC%9F">How Does Context Engineering Actually Work?</h3><figure class="kg-card kg-image-card"><img src="https://media.tenor.com/mIskF4eUG7kAAAAC/sw.gif" class="kg-image" alt="" loading="lazy" width="498" height="373"></figure><p>Let's walk through a concrete example to illustrate how Context Engineering works in practice:</p><p><strong>Our goal:</strong> "Plan a 3-day, 2-night family trip to Tokyo for two adults who love food and anime."</p><p>The AI Agent's thinking and action process is essentially an autonomous reasoning loop centered on managing context:</p><ol><li><strong>Initial Context:</strong> The Agent's working memory now holds our goal: <code>{destination: Tokyo, duration: 3 days / 2 nights, travelers: 2, interests: food, anime}</code>.</li><li><strong>Action 1 (Tool Use):</strong> The Agent decides to tackle transportation first. It runs <code>web_search("Taipei to Tokyo round-trip airfare prices")</code>.</li><li><strong>Context Update:</strong> The tool returns results showing budget airline round-trip tickets starting around NT$8,500. The Agent adds this to its context: <code>{...previous info..., airfare budget: TWD 17,000}</code>.</li><li><strong>Action 2 (Reasoning + Tool Use):</strong> Based on the updated context, the Agent estimates the remaining budget for accommodation and activities, then runs another search: <code>web_search("Tokyo Shinjuku anime-themed hotel double room price")</code>.</li><li><strong>Continuous Iteration:</strong> The Agent keeps repeating the loop — "reason based on current context → decide on an action → use a tool → update context with new results" — until the full itinerary is complete.</li></ol><p>Throughout this process, every prompt is dynamically generated. Managing and optimizing this ever-changing stream of <strong>conversation history + tool results + intermediate reasoning</strong> is the core of Context Engineering.</p><h2 id="%E7%82%BA%E4%BB%80%E9%BA%BC%E6%88%91%E5%80%91%E9%9C%80%E8%A6%81-context-engineering%EF%BC%9F">Why Do We Need Context Engineering?</h2><figure class="kg-card kg-image-card"><img src="https://media.tenor.com/pDxolQOoNk4AAAAC/budget-money.gif" class="kg-image" alt="" loading="lazy" width="200" height="150"></figure><h3 id="%E6%B3%A8%E6%84%8F%E5%8A%9B%E9%A0%90%E7%AE%97%EF%BC%88attention-budget%EF%BC%89">The Attention Budget</h3><p>Just as the human brain has a limited <a href="https://www.sec.ntnu.edu.tw/uploads/asset/data/62563fea381784d09345ba54/02-12-106006-(%E7%9F%A5%E8%AD%98)%E7%B0%A1%E4%BB%8B%E5%B7%A5%E4%BD%9C%E8%A8%98%E6%86%B6%E5%8F%8A%E5%85%B6%E8%85%A6%E5%85%A7%E6%A9%9F%E5%88%B6(%E7%AB%A5%E4%BF%AE).pdf">working memory capacity</a>, LLMs operate under a similar constraint. Context should be treated as a finite resource — and one that exhibits <strong>diminishing returns as its size grows</strong>.</p><p>This dynamic gives rise to a concept called <a href="https://research.trychroma.com/context-rot">Context Rot</a>: as the token count increases, a model's ability to accurately recall information from within its context degrades significantly.</p><p>This effect stems from two underlying factors:</p><ul><li>The Transformer architecture powering LLMs:<ul><li>Every token attends to every other token</li><li>This consumes compute at O(n²) complexity</li></ul></li><li>LLM attention patterns shaped by training data distribution:<ul><li>Shorter sequences are far more common in training data than long ones</li><li>Models have less experience — and fewer dedicated parameters — for handling dependencies that span a long context</li></ul></li></ul><p>This is precisely why Context Engineering matters:</p><blockquote>Precisely selecting the minimum number of tokens with the highest information density, in order to maximize the likelihood of achieving the desired outcome.</blockquote><h3 id="%E4%B8%8D%E5%A6%82%E7%AD%89%E6%9B%B4%E5%A4%A7%E7%9A%84-context-window%EF%BC%9F">Why Not Just Wait for a Bigger Context Window?</h3><figure class="kg-card kg-image-card"><img src="https://media.tenor.com/wDawKwVafzAAAAAC/conan-o-brien-window.gif" class="kg-image" alt="" loading="lazy" width="498" height="266"></figure><p>This might seem like the obvious solution.</p><p>However, for the foreseeable future, context windows of all sizes will remain susceptible to <a href="https://kurtiskemple.com/blog/measuring-context-pollution/">Context Pollution</a> and issues around information relevance.</p><p>Even as model capabilities continue to improve, treating context as a precious and finite resource remains central to building reliable, effective Agents.</p><h2 id="%E5%8F%83%E8%80%83%E8%B3%87%E6%96%99">References</h2><figure class="kg-card kg-bookmark-card"><a class="kg-bookmark-container"
